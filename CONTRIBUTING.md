# 贡献指南

欢迎贡献你自己真正在用的 prompt,或者补充 `domain-adaptation.md` 里的学科适配建议。

## 贡献 prompt

请遵循库里统一的结构,放进 `skill/paper-buddy/references/` 下最贴近的分类文件:

```
# 角色
[这条 prompt 该扮演什么角色/身份]

# 任务
[要完成什么具体任务]

# 要求
1. [具体要求,尽量可执行、可检查]
2. ...

# 输出格式
- [期望的输出结构]

# 提交前自检(可选,但强烈建议)
1. [模型应该在给出答案前自查的点]

# 输入
[占位符,标明用户应该填什么]
```

## 贡献分领域适配建议

如果你在 `domain-adaptation.md` 里没看到自己领域(经济、医学、地学……),欢迎按现有格式加一张卡片:写作特点、常见 venue、推荐模型、调整要点、特别提醒。哪怕只有几条真实经验也很有价值——这部分本来就标注为"高层经验性建议",不要求面面俱到。

## 提交方式

1. Fork 本仓库
2. 新建分支,把改动加到对应的 `references/*.md` 文件里
3. 提 Pull Request,简单说明这条 prompt 是用在什么场景、验证过没有
4. 有想法但还没想清楚具体怎么写?欢迎先开一个 Issue 讨论

## 关于 Skill 部分(`skill/paper-buddy/SKILL.md` 等)

如果你的改动新增了一类子任务(比如加了新的 `references/*.md` 文件),记得同步更新 `SKILL.md` 里的索引表,不然 Claude Code / Codex 不会知道该在什么时候读这个新文件。
