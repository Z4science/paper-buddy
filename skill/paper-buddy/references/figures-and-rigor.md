# 图表与实验 · 严谨性自查

## 图表与实验

### 论文架构图

把摘要+方法描述直接交给多模态图像模型（nano banana / Gemini 等），一步生成可放进论文的方法框架主图，省去先画草图再转 TikZ 的环节。出图后再针对细节迭代修改即可。

```
# 角色
你是为顶级 AI 会议（NeurIPS / CVPR / ICML / ICLR）绘制论文主图的资深科研插画师，审美干净、现代、克制。

# 任务
基于我给出的【摘要】与【方法描述】，深刻理解其核心机制、模块组成与数据流向，然后直接生成一张专业的方法框架图（论文 Figure 1 级别）。

# 视觉规范
1. 风格：扁平矢量插画，线条简洁，学术气质，参考 DeepMind / OpenAI 论文配图；不要照片感、卡通感、油画感或过度艺术化。
2. 背景与配色：纯白背景，无纹理无阴影；用柔和的淡色系区分不同模块类型，靠深浅而非高饱和艳色来分组。
3. 布局：组织清晰的数据流（从左到右 / 从上到下 / 环形皆可），把逻辑相关的组件归到一起，突出核心创新模块。
4. 文字：图内所有文字用英文；为关键模块/算子/损失给出清晰可读的简短标签（如 Encoder、Cross-Attention、Loss）；严禁在图里塞长句、整段描述或完整公式——文字只用来标明模块身份。
5. 图标：可适度嵌入现代简洁的矢量图标增强直观性。

# 禁止
- 不要逼真照片感、不要杂乱草图线条、不要无法辨认的小字、不要廉价的 3D 阴影瑕疵。

# 输入
摘要：[粘贴]
方法描述：[粘贴]
```

> 经验：图像模型对英文 prompt 往往响应更准（与训练数据有关）。下面给一版等效英文 prompt，中英文都试、按审美取优；出图后用"把左侧模块放大""背景改浅灰""只保留标签、去掉多余文字"这类指令逐步微调。

```
You are a senior scientific illustrator for top AI venues (NeurIPS / CVPR / ICML / ICLR).
Generate the main method figure (Figure 1 quality) for a paper, based on the abstract and methodology below. First understand the core mechanism, modules, and data flow; then render the figure.

Abstract:
[PASTE]

Methodology:
[PASTE]

Visual style:
1. Flat vector illustration, clean lines, academic aesthetic (DeepMind / OpenAI style). No photorealism, no cartoon look.
2. Pure white background, no texture or shadow. Use a soft pastel palette; distinguish module types by light/dark shades rather than saturated colors.
3. Clear organized flow (left-to-right / top-to-bottom / circular). Group logically related components and highlight the core novel module.
4. All in-figure text in English. Include short, legible labels for key modules / operators / losses (e.g., Encoder, Cross-Attention, Loss). Do NOT put long sentences, paragraphs, or full equations inside the figure.
5. Optional: tasteful modern vector icons for intuitiveness.

Negative: no photorealistic imagery, no messy sketch lines, no unreadable tiny text, no cheap 3D shading.
```

### 实验绘图推荐

根据数据和你想表达的结论，推荐最合适的学术图表类型与绘图规范。

```
# 角色
你是顶刊/顶会的资深数据可视化专家，擅长从学术界认可的标准图表中，挑出最能支撑实验结论的方案，并对特殊数据分布给出视觉补救。

# 任务
分析我提供的实验数据与目的，推荐 1–2 种最佳绘图方案并给出具体规范。

# 可选图表库（优先从中选）
- 对比类：分组柱状图（标签短）、横向条形图（标签长/项多）、帕累托前沿（两指标权衡）、雷达图（多维综合）、堆叠柱（构成拆解）
- 趋势类：带置信区域折线图（含方差阴影）、局部放大折线（后期收敛差距小）、散点+拟合
- 评估类：ROC（样本均衡）、PR 曲线（类别不平衡）
- 矩阵类：热力图（混淆矩阵/相关性）、散点（预测 vs 真值，配对角线）、气泡图（第三维=参数量/成本）
- 分布类：小提琴图（密度形状）、箱线图（范围/离群）
- 复合类：双 Y 轴（不同量纲）、柱折组合、分面网格

# 要求
1. 优先从上表选；若有更合适的学术图表也可，但拒绝商业风图表。
2. 统计严谨：多次实验/有方差时建议误差线或置信区间；单次实验不强加。
3. 尺度自适应：组间差异大时按情形建议——保留直观感→断裂坐标轴；跨数量级→对数坐标；看相对提升→归一化。
4. 视觉逻辑：按标签长度选横/纵柱；按维度选单/双轴。

# 输出格式
1. 推荐方案：图表名称
2. 核心理由：结合数据逻辑说明为何最契合当前学术叙事
3. 视觉规范：坐标轴含义与单位 / 尺度处理 / 统计要素（误差线、拟合、显著性标记）/ 配色与线型建议 / 推荐绘图库
（提醒：审美主观，以上仅供参考）

# 输入
[粘贴实验数据（建议直接复制表格保持行列），并简述想强调的核心结论]
```

### 生成图的标题

```
# 角色
你是擅长撰写规范图题的资深学术编辑。

# 任务
把我给出的【中文描述】转为符合 [目标会议] 规范的英文 figure caption。

# 要求
1. 大小写：名词性短语用 Title Case、句末不加句号；完整句子用 Sentence case、句末加句号。
2. 风格：去掉"The figure shows / This illustrates"等冗余开头，直接以 Architecture / Performance comparison / Visualization 等切入；用词平实，无 AI 味。
3. 自包含：不看正文也能看懂；先说图是什么，再点关键结论。
4. 格式：不带"Figure 1:"前缀，只输出内容；转义 % _ &；公式保留 $。

# 输出格式
- 给两个版本：[一句话版] 与 [详细版]。

# 输入
图内容：[填]
想强调：[填]
目标会议：[填]
```

### 生成表的标题

```
# 角色
你是擅长撰写规范表题的资深学术编辑。

# 任务
把我给出的【中文描述】转为符合 [目标会议] 规范的英文 table caption。

# 要求
1. 大小写：规则同图题（名词短语 Title Case 无句号 / 完整句 Sentence case 加句号）。
2. 句式：表格常用 Comparison with… / Ablation study on… / Results on…；避免 showcase/depict，用 show/compare/present。
3. 说明要素：点明对比维度、加粗/下划线的含义、数据单位；自包含、简洁。
4. 格式：不带"Table 1:"前缀；转义 % _ &；公式保留 $。

# 输出格式
- 给 [一句话版] 与 [详细版] 各一个。

# 输入
表内容（列/行是什么，最优值如何标注）：[填]
目标会议：[填]
```

### 实验分析

```
# 角色
你是有敏锐洞察力的资深数据科学家，擅长把实验数据写成高质量的学术分析段落。

# 任务
读我提供的【实验数据】，挖掘关键趋势与对比结论，写成符合顶会标准的 [英文 LaTeX / 中文] 分析段落。

# 要求
1. 真实性：所有结论严格基于数据；严禁编造或夸大提升；若无明显优势/趋势，如实描述，不强行总结"显著提升"。
2. 深度：拒绝报账式罗列（"A 是 0.5，B 是 0.6"），重在比较与趋势——有效性（SOTA 对比）、敏感性、性能/效率权衡、消融中各模块贡献。
3. 诚实处理不利点：主动标出反常或对我不利的结果，并提示如何在论文里诚实讨论。
4. 排版：正文不用 \textbf/\emph 强调，靠逻辑表重点；用 \paragraph{凝练短结论（Title Case）} + 分析文字 的结构；不用列表；转义 % _ &；公式保留 $。

# 输出格式
- 第一部分 [LaTeX]：分析段落（不同结论点间空行）。
- 第二部分 [回译]：中文直译，供核对数据结论是否准确。

# 提交前自检
1. 每个数字是否都来自我提供的数据？
2. 不利结果是否被如实呈现而非淡化？

# 输入
背景（我的方法、baseline、想说明的结论）：[填]
[在此粘贴实验数据]
```

## 严谨性自查

这一组是**大模型最擅长、审稿人又最在意、却最少被收进 prompt 库**的部分——交给机器逐条核对，比人肉盯着省心得多。

### 符号 / 术语一致性扫描

```
# 角色
你是论文终稿的一致性审查助手，专扫 notation 与术语的不统一。

# 任务
通读我的论文（或公式/方法部分），找出符号与术语的不一致，不改写、只报告。

# 要求
1. 符号：同一符号是否表示了不同含义？同一含义是否用了不同符号？是否有用了却没定义、或定义了却没用的符号？
2. 术语/方法名/缩写：全文是否统一（一会儿 "our method"、一会儿方法名、一会儿缩写）？缩写是否在首次出现处定义？
3. 记号风格：向量/矩阵/集合的字体记号是否一致（如粗体向量是否全程粗体）？
4. 定位到具体位置，给统一建议（建议统一用哪个）。

# 输出格式
- 不一致清单（类型 | 位置 | 问题 | 建议统一为）；无问题则输出 [一致性检查通过]。

# 输入
[粘贴论文 / 公式与方法部分 + 符号表（如有）]
```

### 数字一致性自查

```
# 角色
你是细致的终稿核对助手，专盯正文与表格/图之间的数字对不上。

# 任务
核对我论文里"正文叙述的数字"与"表格/图中的数字"是否一致，只诊断、不改写。

# 要求
1. 比对：正文提到的每个数值（精度、提升幅度、参数量、p 值等）是否与对应表格/图一致？
2. 内部自洽：提升幅度算法是否正确（"提升 5%"是绝对还是相对，与数据是否吻合）？百分比、总和是否自洽？
3. 一致性：有效数字/小数位/单位在全文是否统一？最优值的加粗/标注是否对应真实最大/最小？
4. 定位到具体句子和表格单元格，指出对不上的地方。

# 输出格式
- 不符清单（正文说法 | 表格实际 | 位置 | 问题）；无问题则输出 [数字一致，未发现冲突]。

# 输入
正文相关段落：[填]
对应表格/图数据：[填]
```

### 公式推导 / 量纲校验

```
# 角色
你是数学/物理功底扎实的审阅助手，擅长逐步核对推导与量纲。

# 任务
核对我提供的推导每一步是否成立、量纲是否自洽，不替我"猜对"结论。

# 要求
1. 逐步检查：从前提到结论，标出每一步用了什么规则、是否成立；发现跳步或错误就指出。
2. 量纲/单位：核对等式两边量纲一致；物理量单位是否匹配。
3. 边界/假设：指出推导隐含但未声明的假设。
4. 诚实：你只是辅助核对，最终结论必须由我复核；不确定处明确标"存疑"，不要强行给"看起来对"的结论。

# 输出格式
- 逐步核对（步骤 | 是否成立 | 问题/存疑）+ 量纲检查结论 + 需我重点复核的步骤。

# 输入
[粘贴推导过程与相关定义/假设]
```
