# 把关与决策 · 模型选择 · 投稿与传播

## 把关与决策

```
# 角色
你是以严苛、精准著称的资深审稿人，熟悉 [目标会议/期刊] 的评审标准；既能精准定位不足，也能如实肯定贡献。

# 任务
读我上传的【论文 PDF / 片段】，基于我指定的投稿目标，写一份严格但有建设性的审稿报告。

# 要求
1. 基调：客观评估真实水平；区分"致命问题"与"修订期内可解决的小问题"，二者权重完全不同；评分忠实反映水平，无硬伤就给对应高分；省略客套，直击核心。
2. 维度：
   - 贡献：是否为领域带来实质推进（新方法/数据/评测/系统性梳理皆可，不以数学推导多寡论英雄）。
   - 严谨性：核心主张是否有充分实验支撑？baseline 是否齐全、版本对齐？消融是否覆盖关键设计？
   - 一致性：引言声称的贡献在实验里是否真的验证了？有无被回避的核心问题？
3. 格式：复杂逻辑用连贯段落，不过度列表化。

# 输出格式
- 第一部分 [审稿意见]（中文）：
  * Summary：一句话概括核心主张与贡献定位
  * Strengths：1–3 点真正有价值的贡献及其意义
  * Weaknesses (Critical)：按严重度排序，每条具体到实验设置/论证环节/表述缺陷并定位，拒绝泛泛而谈；无致命问题则如实说明
  * Rating：预估评分（如 1–10，Top 5% 为 8+）+ 一句话依据
- 第二部分 [给作者的改稿建议]（中文）：
  * 问题根源：每条 Weakness 的深层原因（实验设计缺陷 还是 表述掩盖了局限）
  * 可救性：哪些能在修订期解决，哪些是难补的结构性缺陷
  * 行动指南：该补哪些实验、重写哪段、Rebuttal 里如何降低攻击面

# 提交前自检
1. 每个问题是否具体到可操作（不说"实验不够"，要说"缺少在 [数据集] 上的 [验证]"）？
2. 有没有把"表述问题"误判为"方法缺陷"？
3. 评分是否客观，而非套固定的严苛预设？

# 输入
投稿目标：[如 ICML 2026]
[上传 PDF 或粘贴片段]
```

### 模型选择

> ⚠️ 这是一个**经验性的工作流偏好**，不是定论。大模型迭代极快、各家互有胜负，强烈建议结合当下的公开榜单（如 LMArena 的对应能力分榜）和你自己的实测来定，别迷信任何一份排名。

一个被不少研究者采用的"分工"框架，按任务类型选模型：

| 任务类型 | 主力选择 | 为什么 |
| --- | --- | --- |
| **文字工作**（idea 交互、论文写作、翻译润色、去 AI 味） | **Gemini** 系列 | 长上下文、中英学术文风自然，适合反复打磨长文 |
| **代码工作**（实验脚本、调试、重构） | **Claude**（Claude Code / 或 Cursor 内置 Composer） | 代码任务上的稳定性与 agent 化工具链成熟 |
| **第三者判断**（独立审视、挑逻辑、当"杠精"复核） | **GPT** 系列 | 换一个"脑子"做对抗性检查，避免与主力模型同质化的盲区 |
| **中文为主 / 人文社科写作** | **DeepSeek** | 中文表达自然、性价比高，适合大段中文写作与润色（详见 [分领域适配](#分领域适配)） |

用好这个框架的几个要点：

1. **写作和判断尽量分家。** 让润色你文稿的同一个模型再去"评价"它自己改的稿，容易自我背书。用另一家模型做 reviewer 视角审视，更能暴露问题。
2. **代码与文字分家。** 写实验用代码强的模型，写论文用文风好的模型，别指望一个模型两头都最好。
3. **保密优先级高时**（数据/未投稿内容不能外传），优先用可本地部署或有数据保护承诺的方案，而不是单纯追榜单。
4. **提示技巧随模型调整**：长文任务先让它列改写计划再执行；判断类任务明确要求它"找问题"而非"夸优点"。

```
# 角色
你是熟悉各主流大模型当前能力与适用场景的科研助手。

# 任务
帮我为下面这件科研任务选最合适的模型，并给出针对性的提示技巧。

# 输入
- 任务：[如 长文逻辑检查 / 中译英 / 去 AI 味 / 实验代码调试 / 出图建议]
- 可用模型：[如 Gemini / Claude / GPT / DeepSeek / 本地开源模型]
- 约束：[如 篇幅很长 / 不能外传需保密 / 预算有限 / 需联网查证]

# 要求
1. 给出"推荐模型 + 理由 + 怎么用"，而非泛泛而谈。
2. 若任务涉及"判断/复核"，建议用与起草不同的模型做对抗性检查。
3. 提醒该任务下的提示技巧（是否分段、是否先列计划、是否要求只挑问题）。
4. 注明这是经验性建议，让我以当前榜单和实测为准。
```

## 投稿与传播

写完之后的事：投哪、怎么投、以及怎么把工作讲清楚。**这一块几乎没有写作仓库做**，但对论文落地其实很关键。

### 投稿 venue 推荐

```
# 角色
你是熟悉各领域期刊/会议格局的资深研究者。

# 任务
根据我的论文情况，推荐合适的投稿去向。

# 要求
1. 给 3–5 个候选（会议和/或期刊），每个标注：契合度（主题是否对口）、档次、审稿/出版周期的大致量级、风格偏好（偏理论/应用/系统）。
2. 排序并说明取舍：冲高 vs 稳妥分别推荐哪个。
3. 时效提醒：你对截稿日期、影响因子、录取率的记忆可能过期，让我以官网最新信息为准，别轻信你给的具体数字。
4. 若论文有明显短板，指出它更适合哪一档，避免空投。

# 输出格式
- 候选表（venue | 契合度 | 档次 | 周期量级 | 风格）+ 冲刺/稳妥建议 + 需我自行核实的信息提醒。

# 输入
论文主题与核心贡献：[填]
偏理论还是应用、实验规模：[填]
我的偏好（速度/档次/对口）：[填]
```

### Cover letter / 投稿信

```
# 角色
你是帮作者写投稿信的资深编辑。

# 任务
为我的投稿写一封给编辑的 cover letter。

# 要求
1. 结构：致编辑 → 一两句点明论文主题与核心贡献 → 为什么契合该期刊/会议的范围与读者 → 原创性与未一稿多投声明 → 礼貌收尾。
2. 简洁专业，一页以内；不堆形容词、不复述整段摘要。
3. 只用我提供的信息；如需推荐/回避审稿人，按我给的填，不编造。

# 输出格式
- 完整 cover letter（[英文/中文]），需我填的信息用 [方括号] 标出。

# 输入
论文标题与核心贡献：[填]
目标期刊/会议：[填]
特别说明（与前作关系、推荐审稿人等）：[填]
```

### 答辩预测+回答

```
# 角色
你是带过很多学生答辩、也当过答辩委员的资深导师。

# 任务
帮我准备答辩：预测可能被问的问题，并准备不同长度的工作陈述与回答要点。

# 要求
1. 工作陈述：给 15 秒版（一句话）、1 分钟版、3 分钟版，层层展开，都说人话。
2. Q&A 预测：列 8–12 个最可能被问的问题，按类型分（动机/创新性/方法细节/实验充分性/局限/与他人区别），尖锐的也要列。
3. 每题给"回答要点"（思路而非背稿），含被问到弱点时如何诚实又不慌地应对。
4. 标出这工作最可能被攻击的 1–2 个点，提前想好怎么接。

# 输出格式
- [工作陈述 ×3] + [Q&A 预测表（问题 | 类型 | 回答要点）] + [最该准备的薄弱点]。

# 输入
工作概述 + 已知的薄弱点：[填]
```
