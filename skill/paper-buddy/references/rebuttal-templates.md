# 审稿回复模板

## 审稿回复

> ⚠️ 通用底线（模板里都内置了）：逐条回应不漏项；先复述审稿人关切再回答；不卑不亢，分歧处给证据不嘴硬；改了什么就说改在哪（定位到 Sec./Fig./Table），没真改的别写"已修改"；诚实，不夸大新结果、不藏不利数据。

### 模板 A · 审稿人要"理论支撑 / 解释为什么有效"

审稿人质疑方法缺理论依据、收敛性、或"为什么 work"。最考验功力，目标是给出**诚实且站得住**的理论说明，而不是堆公式糊弄。

```
# 角色
你是一位在 [领域，如机器学习理论 / 数值优化] 有深厚功底的资深研究者，正在帮我回应一条要求"补充理论支撑"的审稿意见。你既懂如何把直觉形式化为命题，也清楚什么时候只能给直觉性论证、不能假装有严格证明。

# 背景输入
- 审稿意见原文：[粘贴]
- 我的方法核心（一句话）：[填]
- 我已有的理论线索 / 假设 / 相关定理：[填，没有就写"暂无"]
- 投稿场景：[会议/期刊 + 是否有附录可放完整推导]
- 回复语言：[英文 / 中文]

# 任务
拆解这条意见，起草一段既严谨又诚实的理论回应，外加可放进正文/附录的理论内容。

# 要求
1. 先定性，再定量：先判断审稿人质疑的是哪一层——没有收敛保证 / 没解释机制 / 缺与已有理论的联系？归类后再答，不要笼统。
2. 论证分级（关键，决定可信度）：
   - 能严格证明：写成"假设…—命题…—证明/证明思路…"，列全所有假设，不藏前提。
   - 只能给直觉：诚实标注为 intuition / informal argument，给机制性解释（为什么这样设计能缓解某个具体问题），尽量与已有理论做类比。
   - 严禁伪造严格性：不要把直觉包装成"定理"，不要写无法证明的强结论。
3. 挂靠文献：指出可引用哪类已有理论来支撑或类比（给方向/关键词，引用我来核实）。
4. 边界诚实：主动说明该理论成立的条件与不成立的情形，把局限写进去。

# 输出格式
- 第一部分 [给审稿人的回复]（[英文/中文]）：复述并感谢关切 → 用 2–4 句给出理论要点（结论+关键假设+直觉）→ 指明完整推导放在了修订版哪里。
- 第二部分 [放进论文的理论内容]：可粘贴的正文/附录文字（命题/假设/证明思路或机制解释），LaTeX 友好。
- 第三部分 [给我的提示]（中文）：列出最可能被继续追问的薄弱点，以及我需要亲自核实的假设或引用。

# 提交前自检
1. 有没有把"直觉"伪装成"定理"？强结论是否都有对应假设？
2. 回复里声称写进论文的内容，第二部分是否真的提供了？
3. 是否诚实标注了理论的适用边界？

# 输入
[在此粘贴审稿意见与上述背景信息]
```

### 模板 B · 审稿人要的实验"能补"

实验你有条件做（有数据、有算力、来得及）。目标是干净利落地承认、补做、客观报告，并把结果接回原有结论。

```
# 角色
你是一位严谨的实验科学家，正在帮我回应一条"建议补充实验"的审稿意见。该实验我已经/即将完成。你的风格是用数据说话，既不夸大也不遮掩不利结果。

# 背景输入
- 审稿意见原文：[粘贴]
- 审稿人想验证的核心问题：[填]
- 新实验设置：[填：数据集/baseline/指标/超参]
- 新实验结果：[粘贴表格或数字；若未跑完写"待补"并给预期]
- 回复语言：[英文 / 中文]

# 任务
起草这条意见的回复，并准备好可放进论文的结果呈现与一段分析文字。

# 要求
1. 态度：开门见山感谢并采纳，明确"我们补做了该实验"，不绕弯子、不先辩解。
2. 结果呈现：整理成清晰对比（小表格或一两句关键数字），标明与原文设置的差异；数字与我提供的一致，严禁编造或往有利方向四舍五入。
3. 诚实解读（关键）：支持原结论就明说"进一步验证了…"；若部分不利或喜忧参半，如实承认并合理解释（不是找借口），说明对主张的影响、是否需调整表述；不利结果不许藏进长句。
4. 接回正文：说明加在了修订版哪个位置，是否因此改了正文表述。

# 输出格式
- 第一部分 [给审稿人的回复]（[英文/中文]）：复述关切 + 致谢采纳 + 关键结果（带数字）+ 一句话结论 + 指明加在何处。
- 第二部分 [放进论文的内容]：可粘贴的结果表/图说明 + 一段分析文字（LaTeX 友好，不加粗）。
- 第三部分 [给我的提示]（中文）：列出我必须先确认为真的事项（实验确实跑了、数字无误、正文确实改了）。

# 提交前自检
1. 报告的每个数字是否都来自我提供的真实结果？
2. 不利或反常结果是否被如实呈现，而非淡化？
3. "已加入 Table X"之类是否对应第二部分真实给出的内容？

# 输入
[在此粘贴审稿意见与上述背景信息]
```

### 模板 C · 审稿人要的实验"补不了"

实验你确实做不了——算力/时间不够、数据拿不到、超出范围。最容易翻车：既不能照做，又不能让审稿人觉得被敷衍。目标是**诚实给理由 + 给出最强替代证据 + 礼貌地重新界定范围**。

```
# 角色
你是一位经验丰富、深谙 rebuttal 之道的资深作者，正在帮我回应一条"我无法照做"的实验建议。你知道：直接拒绝会激怒审稿人，假装答应又会在终稿露馅；正确做法是诚实说明约束、提供力所能及的替代证据、并论证核心主张为何依然成立。

# 背景输入
- 审稿意见原文：[粘贴]
- 我做不了的真实原因：[算力/时间不足 / 数据或代码不可得 / 超出本文范围 / 需数月的新研究]
- 我能提供的替代证据：[小规模代理实验 / 已有相关结果 / 文献旁证 / 理论论证；没有就写"无"]
- 这条建议是否动摇核心主张：[是/否 + 一句话理由]
- 回复语言：[英文 / 中文]

# 任务
起草一段既不照做、又不让审稿人感到被敷衍的回复。

# 要求
1. 先肯定，再说明（顺序不能反）：开头真诚肯定建议价值，让审稿人感到被尊重。
2. 诚实给约束：一句话坦诚为什么这版做不了，给具体原因，不含糊也不卖惨。
3. 给最强替代（关键）：有替代证据就拿出来，说明它如何以更小代价指向同样结论；若建议与核心主张正交，有理有据地论证"本文主张不依赖于此"并定位为未来方向；若确是局限，大方写进 Limitations 并承诺 future work。
4. 控制攻击面：不做兑现不了的承诺，话说到"诚实+已尽力+不影响主结论"为止。
5. 语气：全程尊重、就事论事，绝不流露"你这要求不合理"的情绪。

# 输出格式
- 第一部分 [给审稿人的回复]（[英文/中文]）：肯定建议 → 诚实说明约束 → 给替代证据或正交性论证 → 必要时写入 Limitations/future work。
- 第二部分 [放进论文的内容]（如适用）：可粘贴的 Limitations 段落或范围界定说明。
- 第三部分 [给我的提示]（中文）：评估风险——审稿人可能如何反驳，我还能补什么最低成本的证据。

# 提交前自检
1. 有没有不留情面地拒绝、或流露不满情绪？（有则改成"肯定+约束+替代"）
2. 有没有做出兑现不了的承诺？
3. "做不了"的理由是否具体可信，而非空泛借口？
4. 是否给出了力所能及范围内最强的替代证据？

# 输入
[在此粘贴审稿意见与上述背景信息]
```

### 模板 D · 整封回复信组装器

把用 A/B/C 打磨好的单条回复，拼成完整、格式规范、语气一致、控字数的 response letter。

```
# 角色
你是帮作者统稿 rebuttal 的资深编辑，负责把零散的单条回复整合成一封专业、连贯、控字数的审稿回复信。

# 背景输入
- 各审稿人意见与我已起草的逐条回复：[粘贴，按审稿人分组]
- 字数/篇幅限制：[如"每位审稿人 ≤ 5000 字符"或"无限制"]
- 回复语言：[英文 / 中文]
- 总体基调：[如"诚恳专业"]

# 任务
整合成一封完整的 response letter。

# 要求
1. 结构：开头一段总致谢（≤3 句，可点出本轮整体改动）；按审稿人分组，每条用"Comment（复述/引用）→ Response"逐条回应。
2. 一致性：统一人称（一般用 we）、时态、术语；读起来像一个人一气呵成。跨审稿人的重复关切口径一致，首次详答、后续指回。
3. 篇幅控制：超限时优先压客套和重复，保留实质回应与数据；告诉我每位审稿人当前字符数。
4. 把关：标出任何"声称已修改但我没确认"的地方，提醒我核对，不替我假定已完成。
5. 格式：纯净可粘贴（按目标系统选 LaTeX / 纯文本）；引用审稿人原话可用斜体或引号区分，但不堆格式。

# 输出格式
- 第一部分 [Response Letter]：完整回复信。
- 第二部分 [核对清单]（中文）：逐项列出"需我确认为真"的陈述，以及各审稿人字符数是否在限制内。

# 提交前自检
1. 每条审稿意见是否都回应到了？有无遗漏？
2. 全文人称、时态、术语、口径是否一致？
3. 有没有把未经确认的"已完成"写成既定事实？

# 输入
[在此粘贴各审稿人意见与我的逐条回复草稿]
```
